## 1. 执行原则与 DoD（统一口径）

1. 一次只推进一个主模块，不跨阶段并行开发。
2. 每个模块必须完成闭环：`任务拆分 -> 实现 -> 自测 -> 联调 -> 回归 -> 文档更新 -> Git 提交`。
3. 阶段闸门未通过（特别是测试与提交），不得进入下一阶段。
4. 敏感配置仅允许环境变量注入，禁止硬编码密钥。
5. 当前轮次文档仅更新 3 处：当前阶段文档 + `99-阶段验收清单.md` + `00E-执行总控看板（唯一入口）.md`（状态变更时）。
6. 例外：当本轮是“阶段计划细化执行”（把阶段大任务拆成可验收小任务）且会影响后续阶段的接口/页面占位/数据源口径时，必须同步更新“其他阶段文档（仅依赖相关）”，并确保前后端双端计划都写明（不能只写单端）。
