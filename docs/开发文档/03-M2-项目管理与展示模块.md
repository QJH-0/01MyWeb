# M2 项目管理与展示模块

> 当前状态：进行中（2026-03-24）  
> 说明：后端 CRUD、前台项目展示、后台项目管理页面已具备，待完成联调回归后标记 DoD。

## 阶段目标

实现项目数据的“后台可维护 + 前台可展示”闭环。

## 开发范围

- 前台：项目列表、分类筛选、详情展示、外链跳转。
- 后台：项目 CRUD（标题、简介、技术栈、封面、链接）。
- 文件引用：封面图使用对象存储 URL。

## 数据与接口建议

- 表：`project`（基础信息）、可选 `project_tag`（标签拆分）。
- 接口：
  - `GET /api/projects`
  - `GET /api/projects/{id}`
  - `POST /api/admin/projects`
  - `PUT /api/admin/projects/{id}`
  - `DELETE /api/admin/projects/{id}`

## 任务拆分

1. 设计项目表与 DTO/VO。
2. 实现后端 CRUD 与参数校验。
3. 开发前台列表、筛选、详情页。
4. 开发后台项目管理页。
5. 接入封面 URL 上传/选择流程（先可用 URL，后续接入文件中心）。

## 验收清单

- 后台增删改查正确，分页与筛选可用。
- 前台列表和详情数据一致。
- 无效参数返回统一错误格式。
- 删除或下线项目后前台不可见。

## 阶段完成定义（DoD）

- 项目业务形成完整闭环，关键接口覆盖成功/失败场景。
- 回归不影响 M1 页面访问与导航。
- 通过后进入 M3。
