# 00D-M0 基础框架与联调链路：类设计蓝图（按需读取）

> 何时读取：仅当在 M0 阶段遇到“类名/职责边界不明确、实现落地不确定、或与阶段文档冲突”时读取。  
> 接口契约：URL/鉴权/返回字段以 `API文档与系统架构.md` 为准。

---

## 1. 实现方案（M0）

1. 建立统一响应与统一异常处理。
2. 建立 traceId 注入链路，所有请求可追踪。
3. 健康检查与基础路由联调。
4. JWT 认证与鉴权链路按接口契约落地（详见 `API文档与系统架构.md`）。

---

## 2. 核心类职责（必须存在）

| 类名 | 职责 | 所属包 |
|------|------|--------|
| `ApiResponse<T>` | 统一响应模型 | `common` |
| `GlobalExceptionHandler` | 统一异常到错误码映射 | `common` |
| `TraceIdFilter` | 请求入口生成/透传 traceId | `common` |
| `HealthController` | 健康检查与版本信息 | `controller` |
| `AuthController` | 注册/登录/刷新/登出/当前用户接口 | `controller` |
| `AuthService` | 认证编排 | `service` |
| `JwtTokenProvider` | Access/Refresh token 生成与校验 | `security` |
| `CustomUserDetailsService` | 用户加载与权限映射 | `security` |
| `SecurityConfig` | Spring Security 配置 | `security` |

说明：具体方法签名、参数校验、异常处理细节在阶段文档中定义，避免过早设计。
