## 使用方式

- 每完成一个文档的“阶段完成定义（DoD）”，再进入下一个文档。
- 每个文档都按“任务拆分 -> 实现 -> 自测 -> 联调 -> 回归 -> 提交”执行。
- 未通过测试不得跳阶段。
- 建议先读阶段版，再按对应日程版逐天执行。

## 执行入口（推荐）

1. 打开 `01-M0-基础框架与联调链路.md`，确认当前阶段目标与 DoD。
2. 再打开 `11-M0-日程执行清单.md`，按天拆解并逐项打勾推进。
3. M0 完成并通过测试后，按同样方式进入 `M1`（先阶段版，再日程版）。
4. 任一阶段若未通过 DoD，不得进入下一阶段文档。

## 统一完成标准

- 前后端可启动、核心接口可调用、核心页面可访问。
- 日志无持续报错，关键链路（列表/详情/提交/搜索）可走通。
- 对应阶段测试清单执行通过，才允许 Git 提交。
