# MyWeb 阶段验收清单（闸门版）

> 规则：每个阶段必须“全部验收项通过”后，才允许进入下一阶段。  
> 执行顺序：`M0 -> M1 -> M2 -> M3 -> M4 -> M5 -> M6 -> M7 -> M8`。  
> 通过条件：功能验收 + 测试验收 + 回归验收 + 文档验收 + Git 提交验收全部通过。

## 全局闸门规则（适用于每个阶段）

### A. 功能验收闸门

- [ ] 本阶段功能清单全部完成，无“待补实现”项。
- [ ] 关键用户路径走通（至少 1 条端到端路径）。
- [ ] 无阻断级问题（P0/P1）。

### B. 测试验收闸门

- [ ] 后端测试通过：`mvn test`（输出截图或日志片段）。

- [ ] 本阶段成功/失败/边界用例均已验证并记录（至少 3 个用例）。
- [ ] 测试覆盖率：Controller 层至少 1 个集成测试，Service 层至少 1 个单元测试。

### C. 回归验收闸门

- [ ] 首页、导航、健康检查可用。
- [ ] 已完成阶段核心路径未回退。

### E. 提交验收闸门

- [ ] 本阶段代码已完成 Git 提交（Conventional Commit）。
