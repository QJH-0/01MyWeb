# MyWeb 阶段验收清单（闸门版）

> 规则：每个阶段必须“全部验收项通过”后，才允许进入下一阶段。  
> 执行顺序：`M0 -> M1 -> M2 -> M3 -> M4 -> M5 -> M6 -> M7 -> M8`。  
> 通过条件：功能验收 + 测试验收 + 回归验收 + 文档验收 + Git 提交验收全部通过。

---

## 全局闸门规则（适用于每个阶段）

### A. 功能验收闸门

- [ ] 本阶段功能清单全部完成，无“待补实现”项。
- [ ] 关键用户路径走通（至少 1 条端到端路径）。
- [ ] 无阻断级问题（P0/P1）。
- [ ] **前后端双端交付**：本阶段涉及的后端接口与前端页面/交互均已落地，禁止仅实现单端。
- [ ] **前端降级可用**：当后端 API 连接失败/超时/返回 5xx 时，前端可回退到默认数据渲染且页面有“后端连接失败”可见提示（不白屏、不阻断主路径演示）。

### B. 测试验收闸门

- [ ] 后端测试通过：`mvn -B test`（输出截图或日志片段）。
- [ ] 后端开发启动通过：`mvn spring-boot:run`（dev 环境默认使用 MySQL，输出启动成功日志片段）。
- [ ] 前端构建通过：`npm run build`（输出截图或日志片段）。
- [ ] 本阶段成功/失败/边界用例均已验证并记录（至少 3 个用例）。
- [ ] 测试覆盖率：Controller 层至少 1 个集成测试，Service 层至少 1 个单元测试。

### C. 回归验收闸门

- [ ] 首页、导航、健康检查可用。
- [ ] 已完成阶段核心路径未回退。

### D. 文档验收闸门

- [ ] `开发计划.md` 进度状态已更新。
- [ ] 对应模块文档已更新（`docs/开发文档/0x-*.md`）。
- [ ] 执行入口文档已更新（`docs/开发文档/00E-执行总控看板（唯一入口）.md`，如本轮有状态变更）。

### E. 提交验收闸门

- [ ] 本阶段代码已完成 Git 提交（Conventional Commit）。
- [ ] 提交内容与本阶段范围一致，无跨阶段混入。
- [ ] 提交后可复现启动与核心验证。
