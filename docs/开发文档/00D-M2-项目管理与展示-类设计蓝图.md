# 00D-M2 项目管理与展示：类设计蓝图（按需读取）

> 何时读取：仅当在 M2 阶段需要锁定类名/职责边界时读取。  
> 接口契约：URL/鉴权/返回字段以 `API文档与系统架构.md` 为准。

---

## 1. 实现方案（M2）

1. 管理端创建/编辑项目，先写 Markdown 主源。
2. Markdown 解析后入 MySQL，并写 Outbox 事件。
3. 异步索引到 ES（BM25 + 向量摘要）。

---

## 2. 核心类职责（必须存在）

| 类名 | 职责 | 所属包 |
|------|------|--------|
| `AdminProjectController` | 管理端项目 CRUD | `controller` |
| `ProjectPublicController` | 前台公开项目列表/详情 | `controller` |
| `ProjectService` | 项目业务编排 | `service` |
| `ProjectSyncService` | Markdown 与结构化数据同步 | `service` |
| `ProjectRepository` | 项目持久化 | `repository` |

说明：具体方法签名、参数校验、异常处理细节在 `02-M2` 阶段文档中定义。
