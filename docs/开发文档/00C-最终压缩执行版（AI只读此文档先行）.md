# MyWeb 最终压缩执行版（AI 先读）

> 目的：把所有开发约束压缩为单页执行规则，避免模型自由发挥。  
> 规则强度：最高。与本文冲突时，其他文档必须回写修正。  
> 生效范围：M0~M8 全阶段。

---

## 1. 执行入口（固定）

每次开发回合必须按以下顺序读取并执行：

1. 本文档（`00C`）
2. `00D-全模块实现方案与类设计蓝图.md`
3. `00A-需求基线（AskUserQuestion）.md`
4. `API文档与系统架构.md`
5. 当前阶段文档（`01`~`08`）
6. `99-阶段验收清单.md`

---

## 2. 唯一权威口径（固定）

- **需求范围与边界**：`00A`
- **实现方案与类设计（类名/职责/关键方法）**：`00D`
- **阶段顺序与必做动作**：`00C`（本文件）
- **接口契约（URL/方法/参数/响应/鉴权/分页）**：`API文档与系统架构.md`
- **技术选型与非功能规范**：`tech-spec.md`
- **阶段任务与验收打勾**：`01`~`08` + `99`

冲突处理：按上面顺序高优先级覆盖低优先级。禁止“按当前代码现状自行解释”。

---

## 3. 固定开发顺序（禁止跳阶段）

`M0 -> M1 -> M2 -> M3 -> M4 -> M5 -> M6 -> M7 -> M8`

每阶段固定流程：

`任务拆分 -> 实现 -> 自测 -> 联调 -> 回归 -> 文档更新 -> 阶段提交`

---

## 4. 接口契约硬约束（不可改写）

- 基础路径：`/api`
- 响应包装：`ApiResponse<T>`
- 鉴权方案：`Spring Security + JWT`（`Authorization: Bearer <accessToken>`）
- 权限边界：`/api/admin/**` 需 `ROLE_ADMIN`；`/api/ai/**` 与评论写接口需登录
- AI 流式：`GET /api/ai/chat/stream`（SSE 仅 `delta/done/error`）
- 分页默认：`page=0`、`limit=10`

说明：阶段文档中不得再次定义不同默认值或不同接口方法。

---

## 5. 阶段 DoD（统一）

每阶段必须同时满足：

1. API 可用（契约一致）
2. UI 可用（主路径可达）
3. 数据闭环（写入/读取/边界）
4. 自动化通过（后端 `mvn test`，前端 `npm run build`）
5. 文档更新完成（阶段文档 + `99`）

任一不满足，禁止进入下一阶段。

---

## 6. 明确禁止项（统一）

- 禁止新增未声明业务能力
- 禁止跨阶段混做
- 禁止绕过统一异常与响应框架
- 禁止修改既定接口契约而不先改文档
- 禁止跳过测试/回归直接提交
- 禁止硬编码密钥

---

## 7. 变更流程（固定）

若需改需求、接口、技术方案：

1. 先改文档（`00A/00C/API/tech-spec/阶段文档`）
2. 再改代码
3. 执行测试与回归
4. 更新 `99` 验收结论

未完成步骤 1，不得进入代码实现。

---

## 8. AI 执行指令（可直接引用）

- 只在已声明范围内开发，不做“顺手优化”或“猜测需求”。
- 任何编码前先在 `00D` 锁定类名、职责、关键方法；禁止临时发明万能类。
- 不确定时先以高优先级文档为准，不自行发明字段和接口。
- 输出结果必须能映射到当前阶段清单与验收项。
