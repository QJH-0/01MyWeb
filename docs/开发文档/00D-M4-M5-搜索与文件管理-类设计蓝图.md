# 00D-M4-M5 搜索与文件管理：类设计蓝图（按需读取）

> 何时读取：仅当在 M4/M5 阶段需要锁定类名/职责边界时读取。  
> 接口契约：URL/鉴权/返回字段以 `API文档与系统架构.md` 为准。

---

## 1. 搜索实现方案（M4）

1. 内容变更写 `es_index_outbox`，与业务事务同提交。
2. `SearchOutboxConsumer` 定时消费，失败指数退避重试。
3. 搜索查询统一融合 BM25 与业务排序（时间衰减）。

---

## 2. 搜索核心类职责（必须存在）

| 类名 | 职责 | 所属包 |
|------|------|--------|
| `SearchController` | 搜索 API 出口 | `controller` |
| `SearchService` | 搜索编排层 | `service` |
| `SearchQueryService` | ES 查询和高亮封装 | `service` |
| `SearchOutboxPublisher` | 业务侧发布索引事件 | `service` |
| `SearchOutboxConsumer` | 后台消费索引事件 | `service` |
| `SearchIndexService` | 索引写入/删除/重建 | `service` |

---

## 3. 文件管理实现方案（M5）

1. 文件元数据存 MySQL，文件体走 S3 兼容存储。
2. 下载必须走预签名 URL，避免暴露存储内网地址。
3. 上传严格校验类型、大小、权限。

---

## 4. 文件核心类职责（必须存在）

| 类名 | 职责 | 所属包 |
|------|------|--------|
| `AdminFileController` | 后台文件上传/列表/删除 | `controller` |
| `FilePublicController` | 前台文件下载 | `controller` |
| `FileStorageService` | 文件存储业务 | `service` |
| `ManagedFileRepository` | 文件元数据持久化 | `repository` |
| `ObjectStorageClient` | S3 兼容存储接口 | `storage` |
| `MinioStorageClient` | MinIO 实现 | `storage` |

说明：具体实现细节在 `04-M4` 和 `05-M5` 阶段文档中定义。
