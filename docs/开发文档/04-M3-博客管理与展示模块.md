# M3 博客管理与展示模块

## 阶段目标

实现博客“后台发布 -> 前台展示”的主业务能力，为搜索与 AI 提供内容来源。

## 开发范围

- 前台：博客列表、分页、分类/标签筛选、详情页 Markdown 渲染。
- 后台：博客 CRUD、草稿/发布状态。
- 元信息：发布时间、标签、摘要、阅读信息。

## 数据与接口建议

- 表：`blog`、可选 `blog_tag`、`blog_category`。
- 接口：
  - `GET /api/blogs`
  - `GET /api/blogs/{id}`
  - `POST /api/admin/blogs`
  - `PUT /api/admin/blogs/{id}`
  - `DELETE /api/admin/blogs/{id}`
  - `POST /api/admin/blogs/{id}/publish`

## 任务拆分

1. 完成博客实体、状态机（draft/published）。
2. 后端 CRUD 与发布流程实现。
3. 前台列表、筛选、详情渲染实现。
4. 后台编辑页面接入 Markdown 编辑器。
5. 预留索引双写事件（为 M4 做准备）。

## 验收清单

- 草稿不可在前台展示，发布后可见。
- Markdown 渲染正确，XSS 基础过滤生效。
- 分页、筛选、详情链路稳定。
- 后台操作日志可追踪（至少记录操作类型与目标 ID）。

## 阶段完成定义（DoD）

- 博客主业务可独立运行并具备内容沉淀能力。
- 为 M4 搜索模块准备好可索引数据。
- 通过后进入 M4/M5。
