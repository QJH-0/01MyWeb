# M3 博客管理与展示模块（细化版）

> 约束：本文件仅定义 M3 执行清单；接口字段、响应结构、鉴权与分页口径统一以 `API文档与系统架构.md` 为准。

## 1. 模块目标

实现博客从草稿到发布再到前台展示的完整生命周期，形成可持续沉淀的内容系统。

## 1A. 阶段开发计划（前后端）

### 前端开发计划（M3）

- 前台：`/blog` 列表（分类/标签筛选 + 分页 + 三态），`/blog/:id` 详情（Markdown 渲染 + 元信息 + 评论区）。
- 后台：博客管理（列表 + 新建/编辑 + 发布/取消发布 + 删除），状态与发布时间可见且反馈清晰。
- 评论交互：登录后可评论/回复/点赞；未登录引导登录且不误报成功。
- **后端不可用降级**：前台列表/详情在后端失败时使用默认博客数据渲染并提示“后端连接失败”；评论/点赞等写操作在后端不可用时明确失败提示。

### 后端开发计划（M3）

- 数据与状态机：落地 `blog`、`blog_tag`，实现 `DRAFT <-> PUBLISHED` 状态流转与 `published_at` 规则。
- 管理端接口：`/api/admin/blogs*` CRUD + `publish/unpublish`（鉴权 `ROLE_ADMIN` + `X-Admin-Token`）。
- 前台接口：`/api/blogs`、`/api/blogs/{id}`、`/api/blogs/slug/{slug}` 强制过滤 `status=PUBLISHED AND deleted_at IS NULL`。
- Markdown 安全：渲染/展示链路具备基础 XSS 防护策略（契约与错误码按既定文档）。
- 评论接口：`/api/comments*`（列表公开、写操作需登录），点赞幂等（`comment_id + user_id` 唯一）。
- 自动化：集成测试覆盖“草稿不可见/发布可见/slug 唯一/评论写需登录/点赞幂等”等关键分支（映射到 `99` 的 M3 条目）。

### 联调与验收里程碑（M3）

- 里程碑 1：后台可完成草稿创建、编辑、发布/取消发布与删除。
- 里程碑 2：前台仅展示发布态，详情 Markdown 可阅读，评论链路可用。
- 里程碑 3：构建与测试证据回填到 `99-阶段验收清单.md` 的 `M3` 区域，完成阶段闸门 A~E。

## 3. 状态机

### 3.1 博客状态定义

| 状态 | 值 | 说明 |
|------|-----|------|
| 草稿 | DRAFT | 初始状态，仅管理端可见 |
| 已发布 | PUBLISHED | 前台可见 |

### 3.2 状态流转

```
DRAFT（草稿） --发布操作--> PUBLISHED（已发布）
PUBLISHED（已发布） --取消发布--> DRAFT（草稿）
```

### 3.3 状态规则

- 新建博客默认为 DRAFT 状态
- 首次发布时设置 published_at 时间戳
- 取消发布可重新编辑后再次发布
- 前台接口仅返回 PUBLISHED 状态的博客

## 5. 后端接口与 DTO 契约

### 5.1 前台接口

#### 5.1.1 博客列表

- 接口路径：`GET /api/blogs`
- 请求参数：category（分类筛选）、tag（标签筛选）、page、limit（默认值以 `API文档与系统架构.md` 为准）
- 返回数据：list（博客列表，摘要形式）、total（总数）、page、limit
- 过滤规则：仅返回 status=PUBLISHED 的博客，按 published_at 倒序

#### 5.1.2 博客详情

- 接口路径：`GET /api/blogs/{id}` 或 `GET /api/blogs/slug/{slug}`
- 返回数据：博客完整信息（包含 content、view_count 等）
- 过滤规则：仅返回 status=PUBLISHED 的博客

### 5.2 管理端接口

#### 5.2.1 管理博客列表

- 接口路径：`GET /api/admin/blogs`
- 请求参数：status（状态筛选）、category、page、limit

#### 5.2.2 创建博客

- 接口路径：`POST /api/admin/blogs`
- 请求体字段：title（必填）、slug（必填且唯一）、summary、content、category、coverUrl、tags
- 校验规则：标题必填，slug 必填且唯一，URL 需符合格式

#### 5.2.3 更新博客

- 接口路径：`PUT /api/admin/blogs/{id}`
- 请求体：同创建请求，可部分更新

#### 5.2.4 删除博客

- 接口路径：`DELETE /api/admin/blogs/{id}`
- 软删除：仅更新 deleted_at 字段

#### 5.2.5 发布博客

- 接口路径：`POST /api/admin/blogs/{id}/publish`
- 操作：将 status 改为 PUBLISHED，首次发布设置 published_at

#### 5.2.6 取消发布

- 接口路径：`POST /api/admin/blogs/{id}/unpublish`
- 操作：将 status 改为 DRAFT

### 5.3 错误响应与错误码

- 错误码表与 `ApiResponse.error` 负载语义：见 `错误码与错误响应.md`
- 端点触发条件与返回示例：见下节“端点级 API 细化”

### 5.4 端点级 API 细化（字段约束 / DB 映射 / 错误场景）

> 说明：本节只做“可落地细化”；确定项（URL/方法/鉴权/分页默认/响应外层包裹）仍以 `API文档与系统架构.md` 为准。

#### 5.4.1 `GET /api/blogs`（公开列表）

- 请求参数：
  - `category?`：分类筛选
  - `tag?`：标签筛选
  - `page?`/`limit?`：默认值以 `API文档与系统架构.md` 为准
- 数据过滤口径：
  - `status = PUBLISHED`
  - `deleted_at IS NULL`
  - 标签过滤来自 `blog_tag.tag`
- 响应结构：
  - `ApiResponse<PagedResult<BlogResponseDTO>>`
- `BlogResponseDTO` 映射：
  - `id` -> `blog.id`
  - `title` -> `blog.title`
  - `slug` -> `blog.slug`
  - `summary` -> `blog.summary`
  - `content?` -> `blog.content`（公开详情可能携带内容；此处按 API 返回字段为准）
  - `category?` -> `blog.category`
  - `tags` -> 聚合 `blog_tag.tag`
  - `status` -> `blog.status`
  - `coverUrl?` -> `blog.cover_url`
  - `viewCount?` -> `blog.view_count`
  - `publishedAt?` -> `blog.published_at`
  - `createdAt/updatedAt/deletedAt?` -> 对应时间字段
- 错误场景（示例）：
  - 参数校验失败：`400 VALIDATION_ERROR`
  - 发生未预期错误：`500 INTERNAL_ERROR`

#### 5.4.2 `GET /api/blogs/{id}` / `GET /api/blogs/slug/{slug}`（公开详情）

- 数据过滤口径：
  - `status = PUBLISHED`
  - `deleted_at IS NULL`
- 响应结构：
  - `ApiResponse<BlogResponseDTO>`
- 错误场景（示例）：
  - 草稿/不存在：`404 NOT_FOUND`
  - 发生未预期错误：`500 INTERNAL_ERROR`

#### 5.4.3 `GET /api/admin/blogs`（后台列表）

- 鉴权：需 `ROLE_ADMIN` 且携带 `X-Admin-Token`
- 请求参数：
  - `status?`：DRAFT / PUBLISHED
  - `category?`：分类筛选
  - `page?`/`limit?`：默认值以 `API文档与系统架构.md` 为准
- 数据过滤口径：
  - 默认 `deleted_at IS NULL`
  - status/category 可选筛选
- 响应结构：
  - `ApiResponse<PagedResult<BlogResponseDTO>>`

#### 5.4.4 `POST/PUT /api/admin/blogs`（后台创建/更新）

- 鉴权：同 `5.4.3`
- 请求体（关键约束，按 `API文档与系统架构.md` 为准）：
  - `slug` 必填且唯一
  - `content` 必填，后台发布链路要求 Markdown 安全清洗（阶段约束：只允许安全清洗后的内容入库/或入索引）
- 数据落库口径：
  - 写入 `blog` 基本字段 + `blog_tag` 多行 tag
  - 新建默认 `status = DRAFT`
- 响应结构：
  - `ApiResponse<BlogResponseDTO>`
- 错误场景（示例）：
  - slug 冲突：`400 VALIDATION_ERROR`
  - 权限错误：`401/403`
  - 校验失败：`400 VALIDATION_ERROR`

#### 5.4.5 `POST /api/admin/blogs/{id}/publish`（发布）

- 鉴权：同 `5.4.3`
- 数据落库口径：
  - `status <- PUBLISHED`
  - 首次发布写入 `published_at`（若再次发布不重置，按阶段规则）
- 响应结构：`ApiResponse<BlogResponseDTO>`
- 错误场景：
  - id 不存在/已删除：`404 NOT_FOUND`

#### 5.4.6 评论模块 `GET/POST /api/comments` 与 `reply/like`

- `GET /api/comments`（公开列表）
  - 请求参数：`targetType` + `targetId`，`page/limit`
  - 数据过滤口径：评论按 `target_type/target_id`，并按实现过滤 `deleted_at IS NULL`
  - 响应结构：`ApiResponse<PagedResult<CommentItemDTO>>`
  - `CommentItemDTO` 映射：
    - `id` -> `comment.id`
    - `targetType` -> `comment.target_type`
    - `targetId` -> `comment.target_id`
    - `content` -> `comment.content`
    - `parentId?` -> `comment.parent_id`
    - `likeCount` -> `comment_like` 聚合或 `comment.like_count`
    - `createdAt` -> `comment.created_at`
- `POST /api/comments`（创建评论，登录必需）
  - 错误场景：未登录 `401 UNAUTHORIZED`；target 参数校验失败 `400 VALIDATION_ERROR`
  - 响应结构：`ApiResponse<CommentItemDTO>`
- `POST /api/comments/{id}/reply`（回复，登录必需）
  - 需要校验父评论存在且属于同 target（按实现决定返回码，默认 `404 NOT_FOUND`）
  - 响应结构：`ApiResponse<CommentItemDTO>`
- `POST /api/comments/{id}/like`（点赞，登录必需）
  - 幂等/可切换策略（按实现口径）：若已点赞则取消点赞并返回 `liked=false`
  - 响应结构：`ApiResponse<{ liked: boolean, likeCount: number }>`

## 9A. 固定技术实现方案（禁止自由发挥）

### 9A.1 固定实现顺序

1. 建立博客状态机与表结构（`DRAFT/PUBLISHED`）。
2. 完成管理端 CRUD + 发布/取消发布接口。
3. 完成前台列表/详情接口（仅发布态可见）。
4. 完成 Markdown 渲染与基础 XSS 防护。
5. 完成评论接口（列表/创建/回复/点赞）并联动 blog/project。
6. 完成前后台页面联调与测试回归。

### 9A.2 固定技术方案

- 状态流转：仅允许 `DRAFT <-> PUBLISHED`，不新增第三状态。
- 发布时间规则：首次发布写入 `published_at`，重复发布不重置首次发布时间。
- 前台查询规则：固定过滤 `status=PUBLISHED AND deleted_at IS NULL`。
- 渲染策略：后端存 Markdown 原文，前端渲染并做安全过滤。
- slug 规则：唯一约束 + 后台校验，冲突返回 `BAD_REQUEST`。
- 内容主源：博客内容的权威来源在开发/测试默认为本地 `content/blog/*.md`；生产模式为 GitHub 对应目录。后台创建/编辑/发布最终必须写回 Markdown（见 `tech-spec.md` 3.1A），MySQL/ES 为派生存储与索引，禁止反向覆盖 Markdown。
