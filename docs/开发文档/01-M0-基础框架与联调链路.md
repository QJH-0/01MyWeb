## 3. 接口与契约

### 3.1 健康检查接口

- 接口路径：`GET /api/health`
- 返回内容：mysql、redis、elasticsearch、minio 四个服务的健康状态

### 3.2 统一响应体格式

成功响应包含字段：`success`、`data`、`timestamp`、`traceId`
失败响应包含字段：`success`、`error`（包含 code、message、details）、`timestamp`、`traceId`

### 3.3 错误码规范

| 错误码 | 含义 | HTTP 状态码 |
|--------|------|-------------|
| `BAD_REQUEST` | 参数校验失败 | 400 |
| `UNAUTHORIZED` | 未认证 | 401 |
| `FORBIDDEN` | 无权限 | 403 |
| `NOT_FOUND` | 资源不存在 | 404 |
| `INTERNAL_ERROR` | 系统内部错误 | 500 |
| `SERVICE_UNAVAILABLE` | 服务不可用 | 503 |

### 3.4 统一异常处理

- 参数校验错误：返回 `BAD_REQUEST`，message 包含字段级错误信息
- 业务异常：返回自定义错误码，message 为业务描述
- 系统异常：返回 `INTERNAL_ERROR`，message 为通用提示，details 仅在开发环境包含堆栈

## 8A. 固定技术实现方案（禁止自由发挥）

### 8A.1 固定实现顺序

1. 后端配置分层落地（`application*.yml`）。
2. 中间件连接配置与健康探针实现。
3. 统一响应体、异常处理、traceId、CORS 落地。
4. `GET /api/health` 实现与前端展示联调。
5. 完成测试与文档勾选。

### 8A.2 固定技术方案

- 后端框架：Spring Boot（仅 MVC + Validation + Data/JPA + Filter/Interceptor 能力）。
- 健康检查实现：统一在 `HealthController` 聚合 MySQL/Redis/ES/MinIO 状态。
- 统一响应：所有 controller 返回统一 `ApiResponse`，禁止返回裸字符串或裸 Map。
- 异常处理：统一走 `GlobalExceptionHandler`，禁止在 Controller 内部 try/catch 吞错。
- 链路追踪：使用 `TraceIdFilter` 注入并透传 `traceId`。
- 跨域策略：统一在 `CorsConfig` 配置，禁止在单个接口散落加注解处理。
