### 2.1 工程结构与规范

- 前端目录规范：`src/api`、`src/views`、`src/components`、`src/router`、`src/styles`。
- 后端目录规范：`controller`、`service`、`repository`、`entity`、`dto`、`config`、`common`。
- 统一命名规则：DTO/VO/Entity/Service 后缀一致，接口命名语义清晰。

### 2.2 配置治理

- 后端配置分层：`application.yml`、`application-dev.yml`、`application-test.yml`、`application-prod.yml`。
- 前端环境变量由根目录 `.env`（或 `.env.local`）注入；开发/测试/生产按实际运行 profile 选择。
- 安全要求：密钥仅允许环境变量注入，禁止写入仓库。

### 2.3 基础中间件联通

- MySQL 连接与基础读写验证。
- Redis 连接与 ping 验证。
- Elasticsearch 连通性检测。
- MinIO 连通性检测。

### 2.4 基础后端能力

- 统一响应体：成功/失败结构一致。
- 全局异常处理：参数校验错误、业务异常、系统异常统一返回。
- 请求链路日志：traceId 注入与透传。
- CORS 策略：前端可跨域访问后端 API。

### 2.5 基础联调能力

- 健康检查接口：`GET /api/health`。
- 前端启动后可调用健康检查接口并展示状态。
- Nginx 反代与前端路由回退（SPA）生效。

### 2.6 基础鉴权与认证能力

- JWT 鉴权链路落地：`Spring Security + JWT`，请求可携带 `Authorization: Bearer <accessToken>`。
- 认证接口可用：`POST /api/auth/register|login|refresh|logout`，以及 `GET /api/auth/me`。
- 注册防滥用：实现 IP 频控 + `captchaToken` 最小校验回退（可配置真实校验）。
  - `captchaToken` 最小校验失败 与 IP 频控命中：返回 `400`，`ApiResponse.error` 使用错误码 `VALIDATION_ERROR`（禁止引入新错误码）。
- 权限边界生效：`/api/admin/**`、`/api/ai/**` 与评论写接口需登录/管理员权限。

## 8A. 固定技术实现方案（禁止自由发挥，保留）

### 8A.1 固定实现顺序

1. 后端配置分层落地（`application*.yml`）。
2. 中间件连接配置与健康探针实现。
3. 统一响应体、异常处理、traceId、CORS 落地。
4. JWT 鉴权链路与认证接口联调（register/login/refresh/logout/me）。
5. `GET /api/health` 实现与前端展示联调。
6. 完成测试与文档勾选。

### 8A.2 固定技术方案

- 后端框架：Spring Boot（仅 MVC + Validation + Data/JPA + Filter/Interceptor 能力）。
- 健康检查实现：统一在 `HealthController` 聚合 MySQL/Redis/ES/MinIO 状态。
- 统一响应：所有 controller 返回统一 `ApiResponse`，禁止返回裸字符串或裸 Map。
- 异常处理：统一走 `GlobalExceptionHandler`，禁止在 Controller 内部 try/catch 吞错。
- 链路追踪：使用 `TraceIdFilter` 注入并透传 `traceId`。
- 跨域策略：统一在 `CorsConfig` 配置，禁止在单个接口散落加注解处理。
