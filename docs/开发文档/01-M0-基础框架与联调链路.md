# M0 基础框架与联调链路

## 阶段目标

建立可运行的全栈骨架，打通前后端与基础中间件，让项目具备持续迭代能力。

## 开发范围

- 前端：Vue3 + Vite + Router + Pinia + Axios + SCSS 变量体系。
- 后端：Spring Boot 3.4 + Java 21 + 分层目录结构。
- 中间件连通：MySQL / Redis / Elasticsearch / MinIO。
- 基础部署：Docker Compose + Nginx。
- 基础规范：统一响应体、错误码、日志、CORS、环境变量管理。

## 任务拆分

1. 初始化前后端工程与基础目录。
2. 补齐本地与开发环境配置（`.env.example`、`application-dev.yml`）。
3. 建立健康检查接口与前端示例 API 调用。
4. 编写 `docker-compose`，启动 MySQL/Redis/ES/MinIO/Nginx。
5. 落地全局错误处理和统一响应格式。
6. 完成首次联调（前端页面调用后端健康接口）。

## 验收清单

- 前后端均可本地启动。
- `docker-compose` 一键拉起基础服务。
- 前端可成功调用后端 API，且跨域策略正确。
- 统一响应体与异常处理生效。
- 日志可定位请求链路（至少包含 traceId 或请求标识）。

## 阶段完成定义（DoD）

- 提交可复现的启动步骤（README 或脚本）。
- 新成员可在 30 分钟内完成本地启动。
- 通过后才允许进入 M1。
