# MyWeb 需求基线（AskUserQuestion 访谈版）

> 版本：v1.2  
> 产出日期：2026-03-24  
> 说明：本基线来自“结构化问答”结果，用于约束后续模块开发与验收。若后续需求变化，请直接更新本文档并记录版本。
> 实现落地：对应类名、职责、关键方法统一以 `00D-全模块实现方案与类设计蓝图.md` 为准。

---

## 1. 项目目标与用户定位

### 1.1 核心目标

- 项目首要目标：**持续发布技术内容并沉淀影响力**。

### 1.2 目标用户

- 首要目标用户：**技术同行/开发者**。

---

## 2. 首版范围（MVP Scope）

首版必须包含以下模块（全部进入开发范围）：

- `M1` 首页与个人介绍
- `M2` 项目展示（含后台管理）
- `M3` 博客展示（含后台管理）
- `M4` 全站搜索
- `M5` 文件管理/下载
- `M6` AI 助手（RAG + 会话上下文）
- `M8` 部署上线能力（单机云服务器）

说明：后台能力为首版必需项，前台默认公开访问。

---

## 3. 开发顺序与执行策略

### 3.1 固定顺序

- 按既定顺序推进：`M0 -> M1 -> M2 -> M3 -> M4 -> M5 -> M6 -> M7 -> M8`。

### 3.2 执行原则

- **一个模块一个模块开发**，每个模块必须达到“可跑通、可验收”后再进入下一模块。
- 不做跨模块并行大规模开发，避免收口困难。

---

## 4. 每模块最低验收标准（统一 DoD）

每个模块完成时，必须同时满足：

1. **接口可用**：核心 API 可调用，契约符合预期。
2. **页面可用**：前端主流程可访问、可操作、无阻断。
3. **数据闭环**：数据库读写链路完整，可用假数据验证。
4. **自动化测试**：该模块有基础测试并通过。
5. **文档更新**：阶段文档与进度状态同步更新。

任一项未通过，视为该模块未完成。

---

## 5. 关键能力边界

### 5.1 认证与权限

- 前台：公开访问（无需登录）。
- 后台：需要登录（管理员能力）。

### 5.2 搜索范围

- 首版要求：**全站搜索**（至少覆盖博客、项目、文件元数据）。

### 5.3 AI 助手范围

- 首版要求：**RAG + 会话历史上下文**。
- 至少支持连续对话与来源引用。

### 5.4 假数据策略

- 开发与联调阶段采用：**SQL Seed 脚本**（可重复执行）。

### 5.5 部署目标

- `M8` 优先目标：**单机云服务器 + Docker Compose**。

---

## 6. 关键决策（第三轮补齐）

### 6.1 内容权威来源（最终）

- 最终采用：**混合模式（Markdown 主，数据库做索引/缓存）**。
- 约束要求：
  - Markdown 为内容真源（Source of Truth）。
  - Markdown 主源固定存放在 GitHub 仓库 `content/` 目录（见 `tech-spec.md` 3.1A）。
  - 数据库存储用于检索加速、列表展示与服务化读取，不反向覆盖 Markdown。
  - 同步失败可重试，允许最终一致，但需提供可观测日志。

### 6.2 发布流转

- 首版采用：**草稿/发布两态**。
- 不要求首版实现：审核流、定时发布、版本回滚。

### 6.3 AI 回答与引用策略

- 回答策略：**优先知识库，不足时可补充通用知识并明确标注**。
- 引用最小要求：**来源标题 + 可访问链接**。

### 6.4 搜索排序策略

- 默认采用：**混合排序（相关性 + 时间衰减）**。

### 6.5 非功能目标（首版）

- 核心查询接口：`P95 < 2s`。
- 可用性目标：`99.0%`。
- 必须具备：日志/指标/错误追踪基础能力。

### 6.6 生产数据保护

- 首版最低要求：**手工备份流程先落地，后续再自动化**。

### 6.7 索引/同步实施细则

- 同步触发：仅在后台“发布/更新内容”时触发同步（受控）。
- 失败策略：同步失败自动重试 3 次（指数退避），仍失败则记录待补偿任务（不中断主流程）。
- 可见性口径：同步后尽快刷新，目标在约 `10s` 内可被搜索/检索到（以实现为准，但需满足 DoD 可用性验收）。

### 6.8 AI 通用知识补充规则

- 在允许补充通用知识时，允许“混合回答 + 必须带免责声明”。
- 当提供通用知识时采用：**脚注形式**标注 `（来源：通用公开信息）`。
- 优先在能找到公开来源时附加来源链接；若无法保证来源可靠，保持免责声明并不强行编造链接。

### 6.9 搜索时间衰减参数（混合排序的一部分）

- 默认时间衰减模型：**指数衰减**，半衰期 `30 天`。
- 时间字段优先：使用 `updated_at`（编辑后提升权重）。

## 7. 里程碑验收口径（按用户确认）

- 模块必须逐个通过，不允许“阶段未闭环即切换下一阶段”。
- 验收关注顺序：`API -> UI -> DB -> Test -> Docs`。
- 允许使用假数据推动开发，但必须保证后续可替换为真实数据。
