# M6 AI 助手与 RAG 模块

## 阶段目标

实现可流式对话、可引用来源、可持续维护的 AI 助手能力。

## 开发范围

- 对话模型：DashScope（Spring AI Alibaba）。
- SSE 接口：`delta` / `done` / `error` 事件。
- RAG：向量检索（主）+ BM25（辅）+ RRF 融合。
- 会话存储：Redis（短期）+ MySQL（长期）。

## 任务拆分

1. 接入对话模型与密钥管理。
2. 实现 SSE 流式接口与前端增量渲染。
3. 构建知识库入库流程（公开内容切块、向量写入）。
4. 实现检索融合（vector + BM25 + 去重 + 截断）。
5. 输出引用字段（`sourceType/sourceId/title/url`）。
6. 建立向量重建任务（模型或维度变更可全量重建）。

## 验收清单

- 对话可稳定流式输出并正确结束。
- 回答可携带可跳转引用来源。
- 知识库仅包含公开内容，私有数据不入库。
- Redis 会话过期与 MySQL 持久化策略生效。
- 异常时返回可识别错误事件，不出现静默失败。

## 阶段完成定义（DoD）

- AI 问答形成“可解释、可追溯、可运维”最小闭环。
- 日常问题可命中站内内容并给出来源链接。
- 通过后进入 M7。
