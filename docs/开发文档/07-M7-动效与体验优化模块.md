# M7 动效与体验优化模块

## 阶段目标

在不牺牲可读性和性能的前提下，提升站点观感和交互体验。

## 开发范围

- 路由过渡、滚动触发、卡片悬浮等轻量动效。
- 移动端适配与断点优化。
- 性能优化：懒加载、静态资源压缩、关键请求优化。

## 任务拆分

1. 统一动效时长、缓动函数和触发条件。
2. 完成核心页面响应式适配（首页/博客/项目/AI 入口）。
3. 对图片和重组件进行懒加载。
4. 优化接口请求与错误反馈体验。
5. 做一轮 Lighthouse 或同等指标基线测试。

## 验收清单

- 动效平滑且不干扰阅读。
- 主要页面在移动端无布局断裂。
- 首屏加载与交互延迟有可量化提升。
- 不因动效引入明显 CPU 占用峰值。

## 阶段完成定义（DoD）

- 体验优化结果可复测（有指标或前后对比记录）。
- 通过后进入 M8 上线阶段。
