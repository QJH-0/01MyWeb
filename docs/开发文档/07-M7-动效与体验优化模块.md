# M7 动效与体验优化模块（细化版）

> 约束：本文件仅定义 M7 执行清单；接口字段、响应结构、鉴权与分页口径统一以 `API文档与系统架构.md` 为准。

## 1. 模块目标

提升页面流畅度与交互一致性，在不牺牲稳定性和可读性的前提下优化用户体验。

## 1A. 阶段开发计划（前后端）

### 前端开发计划（M7）

- 路由过渡统一：在路由容器层实现统一 transition（300ms、ease-in-out、淡入淡出 + 轻位移），禁止页面各自定义。
- 交互一致性：卡片 hover、按钮 active、链接 hover、加载/空/错三态样式统一（复用全局变量与通用 class）。
- 响应式与可访问性回归：375/768/≥1024 断点关键页面（首页/项目/博客/搜索/AI）无严重断裂，点击区域可用。
- 性能与包体：路由懒加载、图片懒加载、代码分割；记录构建体积基线并对异常增长列优化项。
- **后端不可用降级一致性**：确保动效/优化不引入白屏，不破坏既有“默认数据渲染 + 可见提示”的降级策略。

### 后端开发计划（M7）

> 说明：M7 以体验为主，后端仅做必要配合（如缓存策略/压缩/安全头/静态资源策略的服务端配合），具体以 `tech-spec.md` 与部署结构为准。

- 输出可观测数据：必要的性能/错误日志与 traceId 链路保持一致（不改变既定响应/错误结构）。
- 为 M8 预热：梳理需要在生产侧落地的安全头、压缩与缓存策略清单，避免上线阶段集中修改。

### 联调与验收里程碑（M7）

- 里程碑 1：动效与交互统一完成，关键页面回归无阻断错误。
- 里程碑 2：移动端关键页面体验达标，性能基线记录完成。
- 里程碑 3：构建与回归证据回填到 `99-阶段验收清单.md` 的 `M7` 区域，完成阶段闸门 A~E。

### 2.4 性能优化

- 重资源按需加载。
- 减少首屏阻塞资源。
- 记录构建体积预警并制定优化项。

## 3. 动效规范

### 3.1 路由过渡动画

| 场景 | 时长 | 缓动函数 | 效果 |
|------|------|----------|------|
| 页面切换 | 300ms | ease-in-out | 淡入淡出 + 轻微位移 |
| 列表项入场 | 200ms | ease-out | 依次淡入 |
| 模态框 | 250ms | ease-out | 缩放 + 淡入 |

### 3.2 交互反馈

| 场景 | 效果 |
|------|------|
| 卡片悬浮 | 轻微上浮（translateY -4px）+ 阴影加深 |
| 按钮点击 | 缩放（scale 0.98） |
| 链接悬停 | 颜色变化 + 下划线 |

### 3.3 响应式断点

| 断点 | 宽度 | 布局 |
|------|------|------|
| 移动端 | < 768px | 单列， hamburger 菜单 |
| 平板端 | 768px - 1024px | 双列或自适应 |
| 桌面端 | > 1024px | 多列，完整导航 |

## 4. 性能指标

### 4.1 构建指标

| 指标 | 目标值 |
|------|--------|
| 首次内容绘制（FCP） | < 1.5s |
| 最大内容绘制（LCP） | < 2.5s |
| 首次输入延迟（FID） | < 100ms |
| 包体大小（gzipped） | < 500KB |

### 4.2 优化手段

- 路由懒加载：动态 import
- 图片懒加载：loading="lazy"
- 组件按需引入
- CSS/JS 代码分割

## 7A. 固定技术实现方案（禁止自由发挥）

1. 统一路由过渡参数（300ms、ease-in-out、淡入淡出 + 轻位移）。
2. 统一交互反馈（卡片悬浮、按钮点击、链接悬停）。
3. 统一加载态/空态/错误态样式。
4. 完成移动端关键页面适配（首页/项目/博客/搜索/AI）。
5. 完成懒加载和包体记录。

### 7A.2 固定技术方案

- 路由动效：统一在路由容器层实现，禁止页面各自定义不同切换动画。
- 交互样式：统一使用全局样式变量与通用 class，禁止分散硬编码。
- 性能策略：固定启用路由懒加载、图片懒加载、代码分割。
- 质量门槛：构建产物需记录体积，出现显著增长必须附优化项。
