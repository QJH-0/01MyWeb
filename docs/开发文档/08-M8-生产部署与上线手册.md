# M8 生产部署与上线手册

## 阶段目标

将项目从“开发可用”推进到“线上稳定可用”，并具备故障快速回滚能力。

## 上线前门禁（必须全部通过）

- 代码门禁：主干构建通过，核心模块测试通过。
- 配置门禁：生产环境变量完整，密钥不入库。
- 数据门禁：MySQL 迁移脚本可回放，ES 索引模板已就绪。
- 安全门禁：HTTPS、鉴权、限流、上传校验已启用。
- 资源门禁：确认 2C4G 负载风险，必要时独立 ES 或升配。

## 发布步骤（建议）

1. 冻结发布窗口并确认变更范围。
2. 打包前后端镜像并标记版本号。
3. 执行数据库迁移（先备份后变更）。
4. 部署应用与中间件配置（Nginx / App / MySQL / Redis / ES / MinIO）。
5. 执行冒烟验证（首页、项目、博客、搜索、AI、上传）。
6. 开启观测（日志、QPS、错误率、内存、磁盘）。

## 上线后 24 小时观察

- 错误率与响应时延是否持续上升。
- ES 与 JVM 内存占用是否逼近阈值。
- AI 接口限流与费用是否异常。
- 文件上传与访问链路是否有权限错误。
- 搜索索引是否出现明显延迟或积压。

## 回滚策略

- 应用回滚：保留上一版本镜像，支持快速切换。
- 数据回滚：数据库迁移脚本必须提供回退方案。
- 索引回滚：保留旧索引别名，必要时切回旧别名。
- 配置回滚：Nginx 与环境变量保留上一版快照。

## 阶段完成定义（DoD）

- 首次正式发布成功并完成 24 小时稳定观察。
- 关键链路无 P0 故障，回滚演练至少完成一次。
- 项目进入“上线后迭代”状态（按版本节奏继续优化）。
