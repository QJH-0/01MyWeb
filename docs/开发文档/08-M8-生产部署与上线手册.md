# M8 生产部署与上线手册（细化版）

> 约束：本文件仅定义 M8 执行清单；接口字段、响应结构、鉴权与分页口径统一以 `API文档与系统架构.md` 为准。

## 1. 模块目标

完成上线前安全与部署收口，保障系统可发布、可观测、可回滚。

## 1A. 阶段开发计划（前后端）

### 前端开发计划（M8）

- 生产构建与配置：补齐生产环境变量与构建产物校验（`VITE_API_BASE_URL` 等），确保 `npm run build` 产物可部署与可回滚。
- 线上可用与降级：生产环境下当后端不可用时提供默认数据或维护提示页，并明确提示“服务不可用/后端连接失败”（文案与样式全站一致）。
- 冒烟路径：首页/项目/博客/搜索/AI/上传的关键页面可达与核心交互可验证。

### 后端开发计划（M8）

- 生产配置收口：`application-prod.yml` 与环境变量模板完整（DB/Redis/ES/MinIO/JWT/限流/日志），缺失配置 fail fast；密钥禁止入库/入仓库。
- 安全基线：HTTPS、鉴权、限流、上传校验、安全响应头全部生效；错误响应不泄露内部细节。
- 可观测与回滚：日志与指标可用（含 traceId），部署编排与回滚脚本可执行；索引重建入口可用。
- 数据门禁：迁移脚本可执行且可回退；备份与恢复流程可演练。

### 联调与验收里程碑（M8）

- 里程碑 1：生产编排可一键部署，HTTPS 与反代策略正确，SPA 回退生效。
- 里程碑 2：全量冒烟链路通过，监控告警与 24h 观察项可执行并记录。
- 里程碑 3：完成一次回滚演练并沉淀结论，证据回填到 `99-阶段验收清单.md` 的 `M8` 区域。

## 2. 上线前门禁（必须全通过）

### 2.1 代码门禁

- 后端：`mvn test` 通过。
- 前端：`npm run build` 通过。
- 无阻断级缺陷（P0/P1）。

### 2.2 配置门禁

- `application-prod.yml` 完整。
- 环境变量模板完整（数据库、缓存、鉴权、限流、存储）。
- 禁止密钥入库。

### 2.3 数据门禁

- 数据库迁移脚本可执行可回退。
- 搜索索引重建入口可用。

### 2.4 安全门禁

- HTTPS 可用。
- 管理端鉴权可用。
- 限流策略可用。
- 上传类型与大小校验生效。
- 安全响应头生效。

## 3. 生产配置要点

### 3.1 后端配置（application-prod.yml）

需要配置的关键项：
- 数据源：生产数据库连接信息
- Redis：生产 Redis 连接信息
- Elasticsearch：生产 ES 集群地址
- MinIO：生产对象存储配置
- 日志：生产级别和输出目标
- 安全：鉴权、限流、CORS 策略

### 3.2 前端配置

需要配置的环境变量：
- `VITE_API_BASE_URL`：生产环境 API 地址
- `VITE_APP_TITLE`：生产环境应用标题

### 3.3 Nginx 配置

需要配置的关键项：
- HTTPS 证书路径
- 静态资源 gzip 压缩
- API 代理转发
- SPA 路由回退
- 安全响应头（X-Frame-Options、X-XSS-Protection 等）

### 3.4 Docker Compose 生产配置

需要配置的服务：
- MySQL 生产实例
- Redis 生产实例
- Elasticsearch 生产实例
- MinIO 生产实例
- 应用服务
- Nginx 服务

## 4. 发布步骤（标准流程）

1. 冻结发布窗口并确认变更清单。
2. 构建前后端产物并标记版本号。
3. 备份数据库，执行迁移脚本。
4. 部署生产编排（应用、Nginx、依赖服务）。
5. 执行冒烟：首页、项目、博客、搜索、AI、上传。
6. 开启观测：错误率、响应时间、资源占用、日志告警。

## 5. 上线后 24 小时观察项

- 接口错误率是否异常升高。
- 响应延迟是否持续增加。
- JVM/ES 内存是否逼近阈值。
- 文件与搜索链路是否出现积压或失败。
- AI 调用是否触发限流或异常费用。

## 6. 回滚策略

- 应用回滚：回退到上一镜像版本。
- 配置回滚：恢复上一版 Nginx 与环境变量快照。
- 数据回滚：按迁移回退脚本执行。
- 索引回滚：通过别名切换或重建回退。

## 8A. 固定技术实现方案（禁止自由发挥）

### 8A.1 固定上线顺序

1. 冻结变更窗口并确认版本。
2. 执行 `mvn test` 与 `npm run build`。
3. 执行数据库备份与迁移脚本。
4. 部署 Docker Compose 生产编排。
5. 执行冒烟链路（首页/项目/博客/搜索/AI/上传）。
6. 启动监控与 24h 观察。

### 8A.2 固定生产技术方案

- 部署模式：单机云服务器 + Docker Compose（首版固定）。
- 反向代理：Nginx 统一处理 HTTPS、SPA 回退、安全头与 API 转发。
- 安全基线：管理端鉴权、限流、上传校验、安全响应头必须同时生效。
- 配置策略：密钥仅环境变量注入，禁止写入仓库与镜像。
- 回滚策略：镜像回退 + 配置快照恢复 + 数据脚本回退 + 索引重建/别名切换。

### 8A.3 通用约束引用（固定）
