# M1 内容展示模块（细化版）

> 约束：本文件仅定义 M1 执行清单；接口字段、响应结构、鉴权与分页口径统一以 `API文档与系统架构.md` 为准。

## 1. 模块目标

构建稳定、可访问、可扩展的门户展示层，作为后续业务模块的统一入口和用户首屏。

## 2. 功能清单

### 2.1 首页（`/`）

- 个人简介展示区（标题、副标题、简介文本）。
- 入口区块：项目入口、博客入口、AI 入口。
- 推荐内容卡片（可先静态，后续切接口）。

### 2.2 关于页（`/about`）

- 个人介绍、技能标签、技术栈概览。
- 可扩展结构（后续可替换为 CMS 或配置化数据）。

### 2.3 经历页（`/experience`）

- 时间线组件展示教育/工作/项目经历。
- 条目支持时间、标题、描述、标签。

### 2.4 联系页（`/contact`）

- 联系方式信息区（邮箱、社交链接等）。
- 表单区（姓名/邮箱/内容）与基础前端校验。

### 2.5 导航与布局

- 顶部导航可达所有页面，当前路由高亮。
- 通用布局（header/main/footer）统一。

### 2.6 后端内容端点（对接 API / 字段约束 / DB 映射）

> 说明：M1 为门户展示首屏，内容由公开端点提供；端点级字段语义以 `API文档与系统架构.md` 为准，且错误码使用 `错误码与错误响应.md`。

#### 2.6.1 `GET /api/content/home`（公开）

- 请求：无
- 响应结构：`ApiResponse<{ title: string, summary: string, sections: any[], updatedAt: string }>`
- DB 映射（建议落地表：`content_page`）：
  - `title` -> `content_page.title`
  - `summary` -> `content_page.summary`
  - `sections` -> `content_page.sections_json`（解析为数组）
  - `updatedAt` -> `content_page.updated_at`（格式化为 ISO8601）

#### 2.6.2 `GET /api/content/about`（公开）

- 同 `2.6.1`

#### 2.6.3 `GET /api/content/experience`（公开）

- 同 `2.6.1`

### 2.7 内容页数据表结构（建议）

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | BIGINT | 是 | 主键，自增 |
| page_key | VARCHAR(50) | 是 | 页面键：home / about / experience（唯一约束） |
| title | VARCHAR(200) | 否 | 页面标题 |
| summary | TEXT | 否 | 页面摘要 |
| sections_json | JSON | 是 | sections 内容（前端按约定渲染） |
| created_at | DATETIME | 否 | 创建时间 |
| updated_at | DATETIME | 否 | 更新时间 |

索引：page_key（唯一）

### 2.8 错误场景（示例）

- 内容页不存在：`404 NOT_FOUND`
- 未预期依赖异常：`500 INTERNAL_ERROR`

## 3. 路由与组件结构

### 3.1 前端路由

需要配置的页面路由：
- `/` - 首页
- `/login` - 登录页（M0 已完成最小可用联调，M1 仅复用与样式统一）
- `/about` - 关于页
- `/experience` - 经历页
- `/contact` - 联系页
- `/projects` - 项目列表页（M2 入口）
- `/projects/:id` - 项目详情页
- `/blog` - 博客列表页（M3 入口）
- `/blog/:id` - 博客详情页
- `/ai` - AI 助手页（M6 入口）
- `/:pathMatch(.*)*` - 404 页面

### 3.2 组件目录结构

```
src/
├── views/                    # 页面组件
│   ├── HomeView.vue          # 首页
│   ├── AboutView.vue         # 关于页
│   ├── ExperienceView.vue    # 经历页
│   ├── ContactView.vue       # 联系页
│   ├── ProjectsView.vue      # 项目列表页
│   ├── ProjectDetailView.vue # 项目详情页
│   ├── BlogView.vue          # 博客列表页
│   ├── BlogDetailView.vue    # 博客详情页
│   ├── AIView.vue            # AI 助手页
│   └── NotFoundView.vue      # 404 页
├── components/
│   ├── layout/               # 布局组件
│   │   ├── AppHeader.vue     # 顶部导航
│   │   ├── AppFooter.vue     # 底部信息
│   │   └── AppLayout.vue     # 通用布局容器
│   ├── home/                 # 首页组件
│   │   ├── HeroSection.vue   # 首屏介绍区
│   │   └── EntryCards.vue    # 入口卡片组
│   ├── common/               # 通用组件
│   │   ├── Loading.vue       # 加载状态
│   │   ├── EmptyState.vue    # 空状态
│   │   └── ErrorMessage.vue  # 错误提示
│   └── experience/           # 经历页组件
│       └── Timeline.vue      # 时间线组件
└── router/
    └── index.ts              # 路由配置
```

### 3.3 通用布局结构

AppLayout 组件包含三个部分：
- 顶部导航（AppHeader）
- 主内容区域（main content）
- 底部信息（AppFooter）

## 7A. 固定技术实现方案（禁止自由发挥）

### 7A.1 固定实现顺序

1. 路由表一次性固定（含 404）。
2. 统一布局组件 `AppLayout/AppHeader/AppFooter` 落地。
3. 四个展示核心页面（`/` `/about` `/experience` `/contact`）完成静态可用（`/login` 由 M0 提供最小可用）。
4. 首页三入口跳转联通（项目/博客/AI）。
5. 补齐 `Loading/EmptyState/ErrorMessage` 并接入页面。
6. 完成移动端 375px 适配与回归。

### 7A.2 固定技术方案

- 前端框架：Vue3 + Vue Router + SCSS，不引入额外状态库做 M1 展示数据。
- 页面组织：固定在 `views`，布局/通用组件固定在 `components/layout` 与 `components/common`。
- 交互策略：每个页面必须实现加载态/空态/错误态三态，不允许空白页。
- 导航策略：路由高亮统一使用 Router 当前路由匹配，禁止手写 DOM class 切换。
