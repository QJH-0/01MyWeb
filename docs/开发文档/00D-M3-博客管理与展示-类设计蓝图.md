# 00D-M3 博客管理与展示：类设计蓝图（按需读取）

> 何时读取：仅当在 M3 阶段需要锁定类名/职责边界时读取。  
> 接口契约：URL/鉴权/返回字段以 `API文档与系统架构.md` 为准。

---

## 1. 实现方案（M3）

1. 博客状态机固定：`DRAFT -> PUBLISHED -> DRAFT`。
2. 发布时触发结构化同步与索引更新。
3. Markdown 渲染必须经安全清洗器。

---

## 2. 核心类职责（必须存在）

| 类名 | 职责 | 所属包 |
|------|------|--------|
| `AdminBlogController` | 后台博客 CRUD + 发布/下线 | `controller` |
| `BlogPublicController` | 前台博客列表/详情 | `controller` |
| `BlogService` | 博客业务编排 | `service` |
| `BlogStateMachine` | 博客状态流转 | `service` |
| `MarkdownSanitizer` | Markdown 安全清洗 | `service` |
| `BlogRepository` | 博客持久化 | `repository` |

---

## 3. 评论模块核心类职责

| 类名 | 职责 | 所属包 |
|------|------|--------|
| `CommentController` | 评论列表/创建/回复/点赞接口 | `controller` |
| `CommentService` | 评论业务编排 | `service` |
| `CommentRepository` | 评论持久化 | `repository` |

说明：具体实现细节在 `03-M3` 阶段文档中定义。
