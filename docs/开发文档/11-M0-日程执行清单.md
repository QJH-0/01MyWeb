# M0 日程执行清单（5 天）

## Day 1：工程初始化

- 创建前端与后端基础工程。
- 建立目录规范与基础依赖。
- 输出：可启动的空白前端页、后端健康接口占位。
- 退出条件：前后端本地均可启动。

## Day 2：配置与环境变量

- 完成前端 `.env.example` 与后端多环境配置。
- 统一端口与 API 前缀约定。
- 输出：配置模板文档与默认开发配置。
- 退出条件：切换环境不改代码，仅改配置可运行。

## Day 3：中间件联通

- 编写 `docker-compose` 启动 MySQL/Redis/ES/MinIO/Nginx。
- 后端完成基础连接与健康检查。
- 输出：一键启动脚本与联通截图/日志记录。
- 退出条件：所有依赖服务健康。

## Day 4：基础规范落地

- 实现统一响应体、全局异常处理、错误码。
- 接入请求日志与 traceId（或请求 ID）。
- 输出：异常返回示例和日志样例。
- 退出条件：成功/失败响应格式统一。

## Day 5：首轮联调与回归

- 前端页面调用后端健康检查与示例接口。
- 验证 CORS、代理、超时与错误提示。
- 输出：M0 验收记录。
- 退出条件：M0 DoD 全部通过。
