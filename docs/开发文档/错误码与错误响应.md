# 错误码与错误响应

> 作用：统一 `ApiResponse.error` 的错误码语义与 HTTP 状态码对应关系。  
> 端点级错误触发条件、返回示例由阶段细化文档补充，但错误码必须来自本文件。

## 1. `ApiResponse.error` 负载语义

- 类型：`string | null`
- 成功：`error = null`
- 失败：`error` 非空
- 格式（推荐，允许实现只返回其中一种）：
  - 仅返回错误码：`<ERROR_CODE>`
  - 错误码 + 可读信息：`<ERROR_CODE>: <message>`
- 禁止：返回堆栈信息、内部敏感细节

## 2. 错误码表（与 HTTP 对应）

| HTTP 状态码 | 错误码 | 典型触发场景 | `error` 示例 |
|---|---|---|---|
| 400 | `VALIDATION_ERROR` | 参数校验失败（缺失/类型错误/范围错误等） | `VALIDATION_ERROR: q 必填` |
| 401 | `UNAUTHORIZED` | 未携带/无效 JWT、RefreshToken 不可用 | `UNAUTHORIZED` |
| 403 | `FORBIDDEN` | 鉴权通过但权限不足（如未持有 `ROLE_ADMIN` 或缺 `X-Admin-Token`） | `FORBIDDEN` |
| 404 | `NOT_FOUND` | 资源不存在或被软删除/不可见（按端点约定） | `NOT_FOUND` |
| 429 | `RATE_LIMITED` | 限流命中 | `RATE_LIMITED` |
| 500 | `INTERNAL_ERROR` | 未知系统异常/依赖不可用且无法降级 | `INTERNAL_ERROR` |

## 3. 阶段文档如何使用本表

- 阶段文档可以写“端点级触发条件 + 返回示例”，但错误码必须使用本表的值。
- 如确需新增错误码：先回填本文件新增条目，再进入实现与阶段更新（遵循 `00C` 的变更流程）。
