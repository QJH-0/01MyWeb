## 2. 安全回滚指令（按场景选择）

原则：**先有检查点，再谈回滚**。未提交的工作优先用工作区回退；已提交但未推送可 reset；已推送只能 revert。

### 2.1 回滚前必做（避免误删）

- `git status`（确认是否有未提交修改）
- `git diff`（确认回滚范围）

### 2.2 场景 A：未提交（只在工作区/暂存区）

- 丢弃未暂存修改：`git restore .`
- 丢弃已暂存修改：`git restore --staged .` 后再 `git restore .`
- 只回退某个文件：`git restore path/to/file`

### 2.3 场景 B：已提交但未推送（本地安全回退）

- 回退到上一个提交：`git reset --hard HEAD~1`
- 回退到任意检查点提交：`git reset --hard <commitSha>`

### 2.4 场景 C：已推送到远端（禁止改写历史）

- 使用反向提交：`git revert <commitSha>`（需要时可多次 revert）

### 2.5 建议的“检查点提交”命名（便于定位）

- `chore(checkpoint): Mx <任务名> (tests: pass)`
