# M4 + M5 搜索与文件管理模块（细化版）

> 约束：本文件仅定义 M4/M5 执行清单；接口字段、响应结构、鉴权与分页口径统一以 `API文档与系统架构.md` 为准。

## 1. 模块目标

补齐"可检索 + 可引用"能力，让博客与项目数据既能被快速搜索，也能被稳定文件链路支撑。

## 1A. 阶段开发计划（前后端）

### 前端开发计划（M4 搜索）

- 搜索入口与页面（如 `/search`，若路由已预留则补齐 UI）：关键词输入、type 筛选（blog/project）、分页与高亮片段展示。
- 三态与降级：后端/ES 不可用时展示默认结果或不可用视图，并提示“后端连接失败/搜索服务不可用”；不阻断主路径演示。

### 后端开发计划（M4 搜索）

- Outbox 双写：在 blog/project 写库事务中写入 `search_outbox_event`（pending），禁止同步直写 ES。
- 消费与重试：pending -> processing -> completed/failed，失败指数退避重试（3 次）并记录 `error_message`。
- ES 索引：`myweb_content` mapping 创建，支持 title/summary/content 检索与 highlight。
- 接口：`GET /api/search`（公开）、`POST /api/admin/search/consume`、`POST /api/admin/search/rebuild`（后台鉴权）。
- 自动化：覆盖 outbox 生成、消费后可检索、参数校验失败、依赖异常分支。

### 前端开发计划（M5 文件）

- 后台文件管理页：上传（类型/大小校验提示）、列表（分页筛选）、删除操作；展示 `accessUrl` 并支持复制/预览。
- 引用链路：在项目/博客编辑页可选择/粘贴文件 URL（按阶段实现方式最小闭环）。
- **后端不可用降级**：列表页可展示默认文件数据并提示“后端连接失败/存储服务不可用”；上传/删除明确失败，不得误报成功。

### 后端开发计划（M5 文件）

- 元数据表：`managed_file`（或阶段约定表）落地 `storage_key/access_url/deleted_at` 等字段与索引。
- 上传：`POST /api/admin/files/upload`（鉴权），类型/大小校验，写 MinIO，元数据入库并回传 URL。
- 列表/删除：`GET /api/admin/files`、`DELETE /api/admin/files/{id}`（软删；物理删由配置控制，默认不删）。
- 下载：`GET /api/files/{id}/download`（公开下载流，正确 Content-Type/Disposition）。
- 自动化：覆盖非法类型/超限/不存在/删除后不可下载/存储异常等分支。

### 联调与验收里程碑（M4+M5）

- 里程碑 1：搜索 outbox 事件可观测，consume 后可在 ES 命中查询，高亮可用。
- 里程碑 2：上传/列表/删除/下载闭环可用，权限边界正确。
- 里程碑 3：构建与测试证据回填到 `99-阶段验收清单.md` 的 `M4/M5` 区域，完成阶段闸门 A~E。

## 2. M4 搜索模块

### 2.1 功能清单

- M4-1 搜索契约与参数校验
  - `GET /api/search`
  - 参数：`q`、`type`、`page`、`limit`
- M4-2 Outbox 双写
  - 业务写库同事务写入 `search_outbox_event`
- M4-3 消费重试机制
  - 定时消费 pending 事件
  - 失败重试、超限失败标记
- M4-4 搜索索引结构
  - 索引文档字段：sourceType/sourceId/title/summary/content/url
- M4-5 检索输出能力
  - 关键词检索、分页、片段高亮
- M4-6 运维入口
  - 手动消费、索引重建接口

### 2.2 接口清单

- `GET /api/search`
- `POST /api/admin/search/consume`
- `POST /api/admin/search/rebuild`

### 2.3 Outbox 表结构

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | BIGINT | 是 | 主键，自增 |
| aggregate_type | VARCHAR(50) | 是 | 聚合类型：blog、project |
| aggregate_id | BIGINT | 是 | 聚合 ID |
| event_type | VARCHAR(50) | 是 | 事件类型：created、updated、deleted |
| payload | JSON | 是 | 事件载荷（包含索引所需数据） |
| status | VARCHAR(20) | 是 | 状态：pending、processing、completed、failed |
| retry_count | INT | 否 | 重试次数，默认 0 |
| error_message | TEXT | 否 | 错误信息 |
| created_at | DATETIME | 否 | 创建时间 |
| processed_at | DATETIME | 否 | 处理完成时间 |

索引：aggregate_type、aggregate_id、status、created_at

### 2.4 Elasticsearch 索引结构

索引名称：`myweb_content`

| 字段名 | 类型 | 说明 |
|--------|------|------|
| source_type | keyword | 来源类型：blog、project |
| source_id | long | 来源 ID |
| title | text | 标题（分词） |
| title_keyword | keyword | 标题（关键词，用于精确匹配） |
| summary | text | 摘要（分词） |
| content | text | 正文内容（分词） |
| category | keyword | 分类 |
| tags | keyword[] | 标签数组 |
| url | keyword | 访问 URL |
| published_at | date | 发布时间 |
| created_at | date | 创建时间 |
| updated_at | date | 更新时间 |

### 2.5 搜索接口契约

#### 2.5.1 搜索查询

- 接口路径：`GET /api/search`
- 请求参数：
  - q（必填）：搜索关键词
  - type：类型筛选（blog/project），默认全部
- page：页码（默认值以 `API文档与系统架构.md` 为准）
- limit：每页条数（默认值以 `API文档与系统架构.md` 为准）
- 返回数据：list（搜索结果，包含高亮片段）、total、page、limit

#### 2.5.2 手动消费

- 接口路径：`POST /api/admin/search/consume`
- 操作：消费 pending 状态的 outbox 事件

#### 2.5.3 索引重建

- 接口路径：`POST /api/admin/search/rebuild`
- 操作：全量重新构建索引

### 2.6 错误响应与错误码

- 错误码表与 `ApiResponse.error` 负载语义：见 `错误码与错误响应.md`
- 端点级触发条件与返回示例：见下节“端点级 API 细化”

### 2.7 端点级 API 细化（字段约束 / DB 映射 / 错误场景）

#### 2.7.1 `GET /api/search`（公开）

- 请求参数：
  - `q` 必填；缺失/空：`400 VALIDATION_ERROR`
  - `type?`：`blog` / `project`（非法值：`400 VALIDATION_ERROR`）
  - `page?`/`limit?`：默认值以 `API文档与系统架构.md` 为准
- 响应结构：
  - `ApiResponse<PagedResult<SearchItemDTO>>`
- `SearchItemDTO` 映射（索引源来自 ES）：
  - `sourceType` -> `myweb_content.source_type`
  - `sourceId` -> `myweb_content.source_id`
  - `title` -> `myweb_content.title`
  - `url` -> `myweb_content.url`
  - `summary` -> `myweb_content.summary`
  - `highlights?` -> ES highlight 片段（按实现的高亮字段与拼接顺序返回）
- 错误场景（示例）：
  - `400 VALIDATION_ERROR`
  - 未预期依赖异常：`500 INTERNAL_ERROR`

#### 2.7.2 `POST /api/admin/search/consume`（后台）

- 鉴权：
  - 需 `Authorization: Bearer <accessToken>` 且 `ROLE_ADMIN`
  - 需携带 `X-Admin-Token`
- 语义：
  - 处理 pending outbox 事件，并回写状态（processing -> completed/failed）
  - 返回 `processed`：本次成功处理的事件数量
- 响应结构：`ApiResponse<{ processed: number }>`
- 错误场景（示例）：
  - 缺失/无效 JWT：`401 UNAUTHORIZED`
  - 权限不足/缺 `X-Admin-Token`：`403 FORBIDDEN`
  - 处理失败且无法回退：`500 INTERNAL_ERROR`

#### 2.7.3 `POST /api/admin/search/rebuild`（后台）

- 鉴权：同 `2.7.2`
- 语义：
  - 全量重建 ES 索引（或触发别名切换 + 重建，以实现为准）
- 响应结构：`ApiResponse<null>`
- 错误场景（示例）：
  - 依赖不可用：`500 INTERNAL_ERROR`

## 3. M5 文件管理模块

### 3.1 功能清单

- M5-1 元数据建模
  - 文件名、类型、大小、存储键、创建时间
- M5-2 上传能力
  - 类型限制、大小限制、存储写入、URL 回传
- M5-3 列表与删除
  - 分页列表、删除记录与物理文件
- M5-4 下载能力
  - 文件流返回、内容类型设置
- M5-5 引用链路
  - 项目/博客可引用文件 URL（后台可维护）

### 3.2 接口清单

- `POST /api/admin/files/upload`
- `GET /api/admin/files`
- `DELETE /api/admin/files/{id}`
- `GET /api/files/{id}/download`

### 3.3 文件表结构

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | BIGINT | 是 | 主键，自增 |
| file_name | VARCHAR(255) | 是 | 原始文件名 |
| file_type | VARCHAR(100) | 是 | 文件 MIME 类型 |
| file_size | BIGINT | 是 | 文件大小（字节） |
| storage_key | VARCHAR(500) | 是 | MinIO 存储路径 |
| access_url | VARCHAR(500) | 是 | 访问 URL |
| uploaded_by | VARCHAR(50) | 否 | 上传人 |
| created_at | DATETIME | 否 | 创建时间 |
| deleted_at | DATETIME | 否 | 删除时间 |

索引：file_type、created_at

### 3.4 接口契约

#### 3.4.1 文件上传

- 接口路径：`POST /api/admin/files/upload`
- 请求：multipart/form-data
- 参数：file（文件）、folder（可选目录）
- 限制：单文件大小默认 10MB，允许类型：图片、Markdown、文本、PDF
- 返回：文件元数据（id、fileName、fileType、fileSize、accessUrl）

#### 3.4.2 文件列表

- 接口路径：`GET /api/admin/files`
- 请求参数：fileType（类型筛选）、page、limit

#### 3.4.3 文件删除

- 接口路径：`DELETE /api/admin/files/{id}`
- 操作：软删除记录，物理文件保留（可配置是否删除）

#### 3.4.4 文件下载

- 接口路径：`GET /api/files/{id}/download`
- 返回：文件流，设置正确的 Content-Type 和 Content-Disposition

### 3.5 错误响应与错误码

- 错误码表与 `ApiResponse.error` 负载语义：见 `错误码与错误响应.md`
- 端点级触发条件与返回示例：见下节“端点级 API 细化”

### 3.6 端点级 API 细化（字段约束 / DB 映射 / 错误场景）

#### 3.6.1 `POST /api/admin/files/upload`（后台）

- 鉴权：
  - 需 `ROLE_ADMIN` 且携带 `X-Admin-Token`
- 请求：
  - `multipart/form-data`
  - `file` 必填；`folder?` 可选（用于生成 storage_key 前缀）
  - 类型/大小限制：单文件默认 10MB，允许类型见 `3.4.1`
- 响应结构：`ApiResponse<FileItemDTO>`
- `FileItemDTO` 映射（来自 `ManagedFile`/文件表）：
  - `id` -> `ManagedFile`（文件元数据表）的 `id`
  - `fileName` -> `file_name`
  - `fileType` -> `file_type`
  - `fileSize` -> `file_size`
  - `storageKey` -> `storage_key`
  - `accessUrl` -> `access_url`
  - `uploadedBy?` -> `uploaded_by`
  - `createdAt` -> `created_at`
  - `deletedAt?` -> `deleted_at`
- 错误场景（示例）：
  - 非法类型/超限：`400 VALIDATION_ERROR`
  - 权限错误：`401/403`
  - 存储失败且无法回退：`500 INTERNAL_ERROR`

#### 3.6.2 `GET /api/admin/files`（后台列表）

- 鉴权：同 `3.6.1`
- 请求参数：
  - `fileType?` 类型筛选（非法值：`400 VALIDATION_ERROR`）
  - `page?`/`limit?`：默认值以 `API文档与系统架构.md` 为准
- 响应结构：`ApiResponse<PagedResult<FileItemDTO>>`

#### 3.6.3 `DELETE /api/admin/files/{id}`（后台删除）

- 鉴权：同 `3.6.1`
- 删除策略：
  - 仅写入 `deleted_at`（软删除）
- 响应结构：`ApiResponse<null>`
- 错误场景（示例）：
  - id 不存在：`404 NOT_FOUND`

#### 3.6.4 `GET /api/files/{id}/download`（公开下载）

- 请求：路径参数 `id`
- 响应：
  - 返回文件流（Content-Type 来自文件 MIME）
  - 设置 `Content-Disposition: attachment; filename="..."`（文件名来自 `file_name`）
- 错误场景（示例）：
  - id 不存在或已删除：`404 NOT_FOUND`
  - 存储读取失败：`500 INTERNAL_ERROR`

## 6A. 固定技术实现方案（禁止自由发挥）

### 6A.1 M4 搜索固定实现顺序

1. 建立 `search_outbox_event` 表与状态字段。
2. 在博客/项目写库事务中固定写入 outbox 事件。
3. 实现消费任务（pending -> processing -> completed/failed）。
4. 实现 ES 索引写入与查询接口。
5. 实现管理端手动消费与重建接口。

### 6A.2 M4 搜索固定技术方案

- 双写模式：仅允许 Outbox 异步索引，不允许业务写库时直接同步写 ES。
- 重试策略：失败重试 3 次（指数退避），超限标记 failed 并记录 error_message。
- 排序策略：固定“相关性 + 时间衰减”，半衰期 30 天，时间字段 `updated_at`。
- 返回结构：必须包含分页与高亮片段。
- 内容来源：索引源数据来自 GitHub `content/` 下的 Markdown 解析结果（由后台发布/更新触发同步），禁止直接以 MySQL 为权威源生成索引数据。

### 6A.3 M5 文件固定实现顺序

1. 建立文件元数据表与索引。
2. 实现上传（类型/大小校验 + 存储写入 + 元数据入库）。
3. 实现列表与删除（软删除）。
4. 实现下载流接口（正确响应头）。
5. 打通博客/项目引用 URL。

### 6A.4 M5 文件固定技术方案

- 存储协议：固定 S3 兼容接口（MinIO），通过 `storage_key` 管理路径。
- 删除策略：固定软删除记录；物理删除策略仅通过配置控制，默认不物理删。
- 下载策略：固定通过后端转发下载，禁止暴露内部存储私有路径。

### 6A.5 通用约束引用（固定）

- 本阶段“验证方式 / 禁止项 / 提交要求”统一执行 `00C-最终压缩执行版（AI只读此文档先行）.md`，不在本文件重复定义。
