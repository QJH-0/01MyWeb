# M4 + M5 搜索与文件管理模块

## 阶段目标

补齐“内容可检索 + 文件可管理”的基础能力，确保博客与项目可被快速定位和引用。

## M4 搜索模块

### 范围

- Elasticsearch 全文索引（博客/项目）。
- MySQL 权威 + ES 最终一致双写（Outbox/重试）。
- 搜索接口：关键词检索、高亮、分页。

### 任务拆分

1. 建立索引结构与映射模板。
2. 实现写入链路（业务写库 + Outbox 事件）。
3. 实现消费链路（异步写 ES + 重试告警）。
4. 提供重建索引脚本或管理入口。
5. 前台接入搜索页与高亮展示。

### 验收

- 新增/更新内容在可接受延迟内可被检索。
- ES 异常时业务写入不丢，重试可恢复。
- 搜索结果高亮、分页、排序可用。

## M5 文件管理模块

### 范围

- 上传图片/Markdown/文本到 MinIO（S3 兼容契约）。
- 文件列表、删除、业务引用关系管理。
- 对外访问 URL 统一由后端生成。

### 任务拆分

1. 设计文件元数据表（key、bucket、mime、size、source）。
2. 实现上传、删除、查询 API。
3. 实现预签名访问或统一网关下载策略。
4. 在项目/博客后台接入文件选择与引用。

### 验收

- 上传后可访问，删除后引用失效可感知。
- URL 编码规则统一，避免双重编码。
- 文件权限与类型大小校验生效。

## 阶段完成定义（DoD）

- 搜索与文件两条链路均形成可监控、可重试、可恢复能力。
- 通过后进入 M6。
